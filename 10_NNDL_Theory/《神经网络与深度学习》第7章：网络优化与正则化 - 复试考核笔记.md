## 第一阶段：网络优化基础与初始化

### Q1：为什么神经网络的参数不能全部初始化为 0？
**问题**：在深层多层前馈神经网络中，隐藏层的参数不能全部初始化为 0 的原因是什么？
**详细解答**：
这种现象在书中被称为**对称权重现象（Symmetry Weight Phenomenon）**。
如果在多层前馈神经网络中将参数全部初始化为 0：
1. **前向传播**：同一个隐藏层中的所有神经元都会接收到相同的输入信号，从而产生完全相同的激活值。
2. **反向传播**：这些神经元接收到的误差梯度也将是完全相同的，导致它们的权重更新量也一模一样。
**结论**：由于参数始终同步更新，深层神经元之间将**没有区分性**（即多个神经元退化为一个神经元），神经网络就失去了提取复杂特征的能力。因此，必须通过**随机初始化**来“打破对称性（Symmetry Breaking）”。

### Q2：动量法（Momentum）是如何改进标准 SGD 的？
**问题**：标准的 SGD 有什么明显的缺点？动量法是如何利用物理学中的“惯性”来解决这个问题的？
**详细解答**：
1. **标准 SGD 的致命缺点**：
   - 遇到**平坦底部（Flat Minima**或局部鞍点时，梯度接近于0，参数几乎无法更新。
   - 遇到**峡谷地形（Ravines）** 时，会在峡谷两壁之间来回**震荡（Oscillation）**，导致在真正最优解方向上的收敛速度极其缓慢。
1. **动量法的数学本质与物理直觉**：
   - 动量法通过计算近期真实梯度的 **“加权移动平均”（Weighted Moving Average）** 来替代当前时刻的真实梯度。公式：$\Delta\theta_t = \rho\Delta\theta_{t-1} - \alpha g_t$。
   - **加速作用**：当最近一段时间内的梯度方向**一致**时，动量会不断累加，加快收敛。
   - **平抑震荡**：当最近一段时间内梯度方向**不一致**（发生震荡）时，相反方向的梯度会相互抵消，减小更新幅度，起到减速稳定的作用。

### Q3：Adam 优化器的本质与矩估计机制是什么？
**问题**：Adam 本质上结合了哪两种优化算法的优点？它是如何利用一阶矩和二阶矩的？为什么初期要进行偏差修正？
**详细解答**：
Adam（自适应动量估计）本质上结合了**动量法（Momentum）** 和 **RMSprop** 的优点。
1. **一阶矩（First Moment）**：梯度的**指数加权移动平均** $M_t$（类似动量法），代表梯度的均值，用来指导参数更新的方向。
2. **二阶矩（Second Moment）**：**梯度平方的指数加权移动平均** $G_t$（类似 RMSprop），代表梯度的未减去均值的方差，用来对每个参数的学习率进行自适应缩放（除以 $\sqrt{G_t}$）。
3. **偏差修正（Bias Correction）**：在迭代初期，由于 $M_0$ 和 $G_0$ 被初始化为全 0 向量，且衰减率（如 0.9 和 0.99）接近 1，会导致初期估计值严重偏向于 0。因此必须通过 $\hat{M}_t = \frac{M_t}{1-\beta_1^t}$ 和 $\hat{G}_t = \frac{G_t}{1-\beta_2^t}$ 进行修正，放大初期的更新步长。

## 第二阶段：正则化与过拟合

### Q4：为什么 $L_1$ 正则化比 $L_2$ 更容易产生稀疏解？（高频必考）
**问题**：从数学公式或几何直觉的角度精确解释 $L_1$ 和 $L_2$ 正则化产生稀疏性（Sparsity）差异的原因。
**详细解答**：
可以从以下两个角度严谨剖析：
1. **几何直觉（约束空间角度）**：
   - **$L_1$ 正则化**的约束空间是一个**菱形**，在坐标轴上拥有**尖锐的顶点（突角）**。目标函数（如平方损失）的等高线向外膨胀时，极大概率会直接与菱形的**顶点**相交。顶点落在坐标轴上意味着有很多特征的权重为绝对的 0，从而产生**稀疏解**。
   - **$L_2$ 正则化**的约束空间是一个**圆形**，边缘平滑无突角。等高线与圆形相切时，交点大概率不在坐标轴上，因此只会让参数值变小，极少绝对为 0。
2. **数学求导（梯度下降角度）**：
   - **$L_1$ 的导数**在非零点始终为常数（$+1$ 或 $-1$）。这意味着无论权重多小，$L_1$ 惩罚项都会以一个**恒定的力量**将其“拉”向 0，一旦到达 0 就会停在原点。
   - **$L_2$ 的导数**与权重成正比（$2w$）。当权重非常接近 0 时，惩罚梯度变得极其微小，使得权重无限趋近于 0，但很难真正变成 0。

### Q5：Dropout 在训练和测试阶段的数学机制是什么？
**问题**：在训练阶段，Dropout 具体如何操作？在测试阶段，为了保证输出期望一致，需要对权重进行什么数学处理？
**详细解答**：
1. **训练阶段**：设置保留概率 $p$。对于神经层，引入一个**丢弃掩码（Dropout Mask） $m \in \{0, 1\}^d$**，其每个元素独立地从概率为 $p$ 的**伯努利分布**中随机生成。实际操作是将输入 $x$ 与掩码 $m$ 进行**按元素相乘（$m \odot x$）**。
2. **测试阶段（极其重要）**：在测试（预测）阶段，**绝不丢弃任何神经元，所有神经元必须处于激活状态**。
   - **期望对齐**：训练阶段神经元输出的数学期望是 $E = p \cdot x + (1-p) \cdot 0 = px$。为了保证测试阶段的输出期望与训练阶段保持一致（防止输入信号过大导致溢出），必须在测试阶段将该神经元的**权重（或输出）乘以概率 $p$**。

### Q6：为什么说 Dropout 本质上是一种集成学习？
**问题**：从集成学习（Ensemble Learning）视角详细解释 Dropout 的机理。
**详细解答**：
1. **指数级的子网络采样**：在训练阶段，每一次前向传播应用 Dropout，假设网络有 $n$ 个神经元，每一次丢弃本质上都是从原始网络中**采样得到了一个子网络**。理论上，总共可以采样出 **$2^n$ 个不同的子网络**。
2. **参数共享机制（Parameter Sharing）**：每次迭代训练不同的子网络，但这 $2^n$ 个子网络是**共享原始网络参数**的。避免了传统集成学习需要海量显存的问题。
3. **近似的群体决策**：测试阶段保留所有神经元并乘以概率 $p$，在数学上可以近似看作是**集成了指数级（$2^n$ 个）不同子网络的输出结果，并进行了加权平均**，从而极大降低了模型的方差，防止过拟合。

### Q7：早停法（Early Stopping）为什么被视为一种隐式正则化？
**问题**：早停法在本质上为什么可以被视为“隐式的正则化”？它和 $L_2$ 正则化有什么内在相似之处？
**详细解答**：
1. **参数空间约束**：神经网络的参数通常被初始化为非常接近于 0 的小值。每一次梯度下降迭代，参数都会离开初始原点一段距离。**提前停止限制了优化算法的迭代次数**。
2. **与 $L_2$ 等价性**：限制迭代次数，本质上就限制了参数在参数空间中能够走多远，从而**隐式地将参数限制在了一个以原点为中心的邻域内**。这与 $L_2$ 正则化显式地将参数“拉”向原点、限制其范数大小的底层作用机理是非常相似的。

## 第三阶段：归一化技术

### Q8：内部协变量偏移 (ICS) 与 Batch Normalization (BN)
**问题**：什么是 Internal Covariate Shift (ICS)？BN 在每一层内部具体是如何操作的？缩放平移参数 $\gamma$ 和 $\beta$ 有什么作用？
**详细解答**：
1. **ICS 定义**：在训练深层神经网络时，由于底层参数的不断更新，导致随后的上层网络输入数据的分布发生剧烈改变，使得模型难以收敛。这种现象称为内部协变量偏移。
2. **BN 计算范围**：针对网络中间层的**净输入（Net Input） $z^{(l)}$** 的**每一维特征**，在当前的**一个小批量（Mini-Batch）样本**上计算均值 $\mu_B$ 和方差 $\sigma_B^2$，并标准化为标准正态分布。
3. **$\gamma$ 和 $\beta$ 的核心作用**：如果直接将数据强制归一化到标准正态分布，净输入会集中在 0 附近。若使用 Sigmoid 等激活函数，该区域刚好是接近线性的变换区间，会**极大地减弱神经网络的非线性表示能力**。引入可学习的缩放参数 $\gamma$ 和平移参数 $\beta$ ($\hat{z} = z_{norm} \cdot \gamma + \beta$)，能够让网络自行恢复/改变取值区间，保留非线性表达能力。

### Q9：Batch Normalization (BN) 的局限性
**问题**：BN 在什么情况下会失效或者效果很差？
**详细解答**：
1. **Batch Size 过小（统计学视角的致命伤）**：BN 的核心思想是用 Mini-Batch 的统计量来近似全局真实数据的期望和方差。当 Batch Size 过小时（如为 2 或 4），均值和方差的估计会非常不准确，产生极其剧烈的统计震荡（巨大的噪声），导致模型根本无法收敛。
2. **处理变长序列（如 RNN/NLP 任务）**：BN 是在“Batch 维度（样本维度）”上计算的。处理变长序列时，不同样本长度参差不齐，且 RNN 在每个时间步 $t$ 的隐藏状态分布都在剧烈地动态变化。BN 无法对齐各个时刻的特征进行批量统计，Padding 补零也会导致统计量失去意义。

### Q10：Layer Normalization (LN) 的优势及应用场景
**问题**：LN 的计算范围是什么？它和 BN 的本质区别在哪？为什么 Transformer/RNN 中清一色使用 LN 而不是 BN？
**详细解答**：
1. **归一化维度的本质区别**：
   - **BN**：在一个小批量（Batch）样本上，针对**某一维特征（每一行）** 进行归一化。
   - **LN**：针对**单个样本**，对其**所有特征/神经元（每一列）** 进行归一化。
1. **克服 Batch Size 依赖**：LN 的计算完全不需要依赖其它样本，因此完美避开了 BN 在 Batch Size 过小时统计量震荡、导致失效的致命缺陷。
2. **完美契合变长序列**：在 RNN 或 Transformer 中处理变长序列时，LN 是在每个特定的时间步 $t$ 独立对当前单个样本的隐藏状态 $h_t$ 进行归一化，完全不受序列长度不一致及动态计算图的影响。
